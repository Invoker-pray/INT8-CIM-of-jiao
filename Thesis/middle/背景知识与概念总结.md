# 中期答辩背景知识与概念总结

---

## 一、核心概念

### 1. 内存墙（Memory Wall）
- **是什么**：CPU/GPU计算速度提升远快于内存带宽提升，导致处理器大部分时间在等数据，而不是在算
- **在神经网络中的体现**：LeNet-5的权重约170 KB，每次推理都要从内存搬到计算单元，搬运能耗是乘加运算的几十倍
- **本项目的解法**：把权重存在计算单元旁边的SRAM里，减少搬运距离

### 2. 存算一体（Computing-in-Memory, CIM）
- **核心思想**：在存储阵列内部或紧邻存储的位置完成计算，数据不离开存储
- **两种极端**：
  - **模拟CIM**：用SRAM cell的电流做模拟乘加，极低功耗但精度差（受工艺偏差影响）
  - **数字CIM**：权重存在SRAM，读出后用数字乘法器计算，精度高，本项目采用此路线
- **本项目定位**：数字CIM，SRAM存INT8权重，标准数字逻辑做乘加，FPGA验证

### 3. INT8量化
- **为什么量化**：神经网络训练用FP32，但推理时精度要求没那么高，用INT8（-128到127的整数）可以把存储和计算量降低4倍
- **量化公式**：`x_int8 = round(x_float / scale) - zero_point`
- **重量化（Requantization）**：INT8×INT8的乘积是INT16/INT32，需要再压缩回INT8才能送下一层
  - 公式：`output_int8 = clamp(round(acc_int32 × mult >> shift), -128, 127)`
  - `mult`和`shift`是量化参数，由训练后量化工具计算

### 4. 矩阵向量乘（MVM, Matrix-Vector Multiply）
- **神经网络的核心运算**：全连接层 = 权重矩阵 × 输入向量 + 偏置
- **本项目的做法**：把权重矩阵分成16×16的Tile，每个Tile一次做16×16的乘加
- **im2col**：把卷积层转化为矩阵乘法的技巧——把输入特征图的每个卷积窗口展开成一列，卷积核展开成行，变成矩阵乘法后用同一套MVM硬件处理

## 二、硬件设计概念

### 5. FPGA与Zynq-7020
- **FPGA**：现场可编程门阵列，可以用硬件描述语言（Verilog/SystemVerilog）编程，实现任意数字电路
- **Zynq-7020**：ARM处理器（PS，Processing System）+ FPGA逻辑（PL，Programmable Logic）的SoC
  - PS：运行Linux，执行Python驱动代码
  - PL：实现CIM加速器硬件
- **PYNQ-Z2**：基于Zynq-7020的开发板，支持Python编程（PYNQ框架）

### 6. SRAM与BRAM
- **SRAM**：静态随机存储器，速度快，本项目用来存权重
- **BRAM（Block RAM）**：FPGA内置的专用存储块，比用LUT拼出来的存储效率高很多
- **BRAM推断**：Vivado综合工具自动把RTL代码里的存储器映射到BRAM的过程
  - 关键约束：必须整字写入，不能有位选择（bit-select），否则退化为寄存器

### 7. 流水线（Pipeline）
- **是什么**：把一个多步骤的操作拆成多个阶段，每个阶段用一个时钟周期，不同数据可以同时在不同阶段处理
- **本项目的7级流水线**：权重加载→零点减法→MAC→偏置累加→ReLU→重量化→写回
- **时序违例（Timing Violation）**：某个组合逻辑路径太长，在一个时钟周期内算不完，WNS（最坏负裕量）为负
- **C1优化**：把后量化的rounding+barrel shift拆成两个时钟周期，解决时序违例，从60 MHz提升到80 MHz

### 8. AXI总线
- **AXI4-Lite**：ARM公司定义的总线协议，每次传输32 bit，适合寄存器读写（CSR配置）
- **AXI4-Stream**：流式传输协议，没有地址，数据连续流入，适合大批量数据搬运
- **DMA（Direct Memory Access）**：直接内存访问，不经过CPU，由专用硬件把数据从内存搬到目标位置
- **本项目的C3优化**：用Xilinx axi_dma IP把权重从DDR内存通过AXI4-Stream批量送到CIM加速器，绕过逐字MMIO写入

### 9. CSR（Control and Status Register）
- **是什么**：控制状态寄存器，软件通过读写这些寄存器来配置和控制硬件
- **本项目的CSR**：包括输入维度、输出维度、量化参数、启动信号、状态查询等，地址映射在0x40000000

## 三、软件与验证概念

### 10. Golden Model（参考模型）
- **是什么**：用Python写的、与硬件行为完全一致的软件模型，用于验证硬件输出是否正确
- **bit-exact**：硬件输出与Python参考模型的每一个字节完全相同，不是"近似正确"而是"完全一致"

### 11. VCS仿真
- **是什么**：Synopsys VCS是业界标准的RTL仿真工具，在电脑上模拟硬件行为
- **验证流程**：Python生成测试向量 → VCS仿真RTL → 对比输出 → 上板验证

### 12. SQ-mapping（空间展开映射）
- **问题**：卷积层的col_len（输入维度）很小（如25），MAX_IN_DIM=784，硬件利用率低
- **解法**：把多个输出像素的输入拼成一个长向量，权重矩阵沿对角线复制，一次MVM处理多个像素
- **效果**：Conv1的k_pack=21，一次处理21个像素，减少MVM调用次数

### 13. PicoRV32
- **是什么**：开源的RISC-V软核处理器，可以综合到FPGA的PL侧
- **本项目的用途**：替代ARM PS控制CIM推理，实现纯PL侧自主推理（不依赖ARM）
- **意义**：验证CIM IP可以被任意处理器控制，不绑定ARM生态

## 五、答辩时需要能解释的逻辑链

1. **为什么做CIM** → 内存墙问题 → 数据搬运能耗远大于计算 → CIM减少搬运
2. **为什么选数字CIM** → 精度可控 → FPGA可综合 → bit-exact验证
3. **为什么用INT8** → 推理精度够用 → 存储/计算量降4倍 → 量化感知训练保精度
4. **Tile怎么工作** → 16×16乘加 → 256 MAC/cycle → 多Tile并行 → 完整MVM
5. **为什么BRAM要拆16 bank** → 整字写入约束 → 否则退化寄存器 → 资源爆炸
6. **时序违例怎么解** → 关键路径太长 → 流水线拆分 → 每级路径缩短 → 频率提升
7. **为什么DMA只快3.35倍而不是223倍** → 瓶颈转移到Python im2col → 下一步向量化优化
